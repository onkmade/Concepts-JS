# Logical Operators 

There are 4 main logical operators In JavaScript:
1. **AND (`&&`)**: Returns `true` if both operands are true.
2. **OR (`||`)**: Returns `true` if at least one operand is true.
3. **NOT (`!`)**: Returns the opposite boolean value of the operand.
4. **Nullish Coalescing (`??`)**: Returns the right operand if the left operand is `null` or `undefined`; otherwise, it returns the left operand.

## 1. OR Operator (`||`)
The OR operator (`||`) is used to combine two or more boolean expressions. It returns `true` if at least one of the expressions evaluates to `true`. If all expressions are `false`, it returns `false`.
### Syntax
```javascript
expression1 || expression2
```
### Example
```javascript
let isWeekend = true;
let isHoliday = false;
if (isWeekend || isHoliday) {
    console.log("You can relax!");
} else {
    console.log("Time to work!");
}
```

## AND Operator (`&&`)
The AND operator (`&&`) is used to combine two or more boolean expressions. It returns `true` only if all expressions evaluate to `true`. If any expression is `false`, it returns `false`.


### Syntax
```javascript
expression1 && expression2
```
### Example
```javascript
let isWeekend = true;
let isSunny = true;
if (isWeekend && isSunny) {
    console.log("Let's go to the beach!");
} else {
    console.log("Maybe another time.");
}
```

## NOT Operator (`!`)
The NOT operator (`!`) is used to reverse the boolean value of an expression. If the expression is `true`, it returns `false`, and if it is `false`, it returns `true`.
### Syntax
```javascript
!expression
```
### Example
```javascript
let isRaining = false;
if (!isRaining) {
    console.log("It's a sunny day!");
} else {
    console.log("Better take an umbrella.");
}
```

## Nullish Coalescing Operator (`??`)
the **nullish coalescing operator (`??`)** is widely used in modern JavaScript.

It’s especially useful when you want to provide a **default value** for something that might be `null` or `undefined`, without affecting other “falsy” values like `0` or `""`.

Example:

```js
let userScore = 0;
let score = userScore ?? 10;  // score is 0, not 10, because 0 is not null/undefined
```

**Use case:** setting defaults safely while ignoring valid falsy values.
